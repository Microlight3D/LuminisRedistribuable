Complex duplication pattern

In this tutorial you will learn to write a script that duplicates an object in a spiral pattern. Here is the full code bellow:

 1import math
 2from System.Numerics import Vector3, Vector2
 3
 4N = 50 # Number of objects
 5R = 5  # length of the radius from the center of the spiral
 6D = 35 # Approximate distance between objects
 7P = Vector2(150,150) # Spiral center position
 8
 9if(fabrication.Assemblies.Count > 0 and len(fabrication.Assemblies[0].getTasks()) > 0):
10    task = fabrication.Assemblies[0].getTasks()[0]
11    for n in range(1,N+1):
12        K = (2*D)/R
13        copy = task.duplicate()
14        x : float = P.X + R*math.sqrt(K*n)*math.cos(math.sqrt(K*n))
15        y : float = P.Y + R*math.sqrt(K*n)*math.sin(math.sqrt(K*n))
16        copy.Parameters.Position = Vector3(x,y,0)
17        fabrication.Assemblies[0].add(copy)
18        fabrication.addTaskToBlueprint(copy)
19else:
20    raise ValueError("No task to duplicate, please load or slice one.")

Retrieving a task

In order to duplicate a task we need to retreive one. For this exemple we will use the first one we find in the Assembly tab:

1task = fabrication.Assemblies[0].getTasks()[0]

But what happens if no task is present in the Assembly tab ? We get an obscure error about an out of range index. Lets check first if the Assembly tab contains a task and raise a clear error if it doesn’t:

1if(fabrication.Assemblies.Count > 0 and len(fabrication.Assemblies[0].getTasks()) > 0):
2    task = fabrication.Assemblies[0].getTasks()[0]
3else:
4    raise ValueError("No task to duplicate, please load or slice one.")

Duplicate N times

In order to form a pattern we need to duplicate the object a number of times and add the duplicates to Luminis:

1N = 50 # Number of objects
2for n in range(1,N+1):
3    copy = task.duplicate()
4    fabrication.Assemblies[0].add(copy)
5    fabrication.addTaskToBlueprint(copy)

Arrange in a spiral pattern

Now that we have a number of duplicates of the same object we can change their properties however we please. For this exemple we choose to arrange them in a spiral pattern. For this we will need to import some math from python and vectors from C#:

1import math
2from System.Numerics import Vector3, Vector2

To dispose the tasks we’ll use the cartesian definition of a spiral from Wikipedia and modify it, for ease of use, to get a distance d between objects instead of an angle \({\phi}\). We’ll also add a center position P for the spiral:

1N = 50 # Number of objects
2R = 5  # length of the radius from the center of the spiral
3D = 35 # Approximate distance between objects
4P = Vector2(150,150) # Spiral center position
5
6for n in range(1,N+1):
7    K = (2*D)/R
8    x : float = P.X + R*math.sqrt(K*n)*math.cos(math.sqrt(K*n))
9    y : float = P.Y + R*math.sqrt(K*n)*math.sin(math.sqrt(K*n))

Finaly we can simply set the computed position to each task copies:

1copy.Parameters.Position = Vector3(x,y,0)

If we put all these steps in order we get our final script:
